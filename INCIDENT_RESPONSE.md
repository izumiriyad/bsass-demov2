# Incident Response

## Severity

- SEV1: Site unavailable, wallet unavailable, authentication broken.
- SEV2: Payment degradation, provider issues, major UX regression.
- SEV3: Cosmetic bug, isolated route issue, content issue.

## Process

1. Acknowledge incident.
2. Identify impacted routes/providers.
3. Enable maintenance messaging if needed.
4. Roll back or patch.
5. Communicate status updates.
6. Write postmortem with prevention actions.
